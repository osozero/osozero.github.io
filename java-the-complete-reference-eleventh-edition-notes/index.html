Java The Complete Reference 11th Edition Notes

 Posted on March 26, 2019  |  7 minutes

Recently, i have read Java: The Complete Reference 11th Edition to refresh my memory. I’ve written some notes and some code samples to clarify related note. You can read my notes and inspect related code samples. Have fun!

  • When an arithmetic operation is being done, all primitive types in the operation are promoted to the biggest type automatically.
  byte b =42;
  char c ='a';
  short s =1024;
  int i=5000;
  float f = 5.67f;
  double d=.1234;

  int result = b*c/s; // b*c is promoted to int, b*c/s is also promoted to int.

  int anotherResult = (f*b) + (i/c)-(d*s); // b is promoted to float because of f is float. c is promoted to int, then s is promoted to double and all result is promoted to double.

  • The elements in the array allocated by new will automatically be initialized to zero (for numeric types), false (for boolean) and null (for reference types).

  • Although you can declare an array type using var, you cannot use var with an array initializer.

  var myArray = new int[5];// valid
  var myArray = {1,2,3,4,5}; //error

  • main method is static because it must be called before any object exist.

  • If you need to compute in order to initialize static variables, you can declare a static block that gets exactly only once, when the class is first loaded.

  class MyClass{
    static int a = 4;
    static int b;

    static{
      System.out.println("Static block initialized");
      b = a+5;
    }
  }

  • As soon as the MyClass class is loaded all static statements are run. First an initialized and assigned to 4. Then static block is executed and then if there is any static block or variable below this block it is executed. Static statements are run by their declaration order. (a is run before b, b is run before static block etc.)

  • Non static inner class can access all member of its outer class directly But reverse is not true.

  class Outer{
    int age;

    class Inner{
      int salary;

      void printAge(){
        System.out.println(age);//can access outer's age.
      }
    }

    void printSalar(){
      System.out.println(salary);//error can't access inner's salary directly.
      Inner in = new Inner();
      System.out.println(in.salary);//can access over instance of inner object.
    }
  }
  • Static inner class can access all member of its outer class over outer’s instance.
  class Outer{
    String name;

    static class Inner{
      int id;

      void changeName(String newName){
        name = newName//error. can not access directly.

        Outer outer = new Outer();
        outer.name = newName;//OK, can access over instance.
      }
    }
  }
  • It is also possible to create nested class in a block (even if in a for loop)
  class Outer{
    int x = 10;

    void test(){
      for(int i=0;i<10;i++){
        class inner{
          void display(){
            System.out.println("display from inner, x: "x);
          }
        }
        Inner in = new Inner();
        inner.display();
      }
    }
  }

  • super() always point the closest superclass.

  • Any class contains one ore more abstract method must be declared as abstract class.

  • There can be no object of any abstract class. Because abstract class is not fully implemented.

  • Concrete methods (implemented) are still allowed in abstract class.

  • Although abstract class cannot be used to instantiate objects, it can be used as a reference to reference subclass of its own.

  abstract class Shape{
    abstract double area();

    void printClasName(){
      System.out.println("Shape");
    }
  }

  class Triange extends Shape{
    double area(){
      System.out.println("area calculation for triangle");
    }
  }

  static void main(){
    Shape s=new Shape();//error abstract class can not be initialized

    Triangle t = new Triangle();

    Shape s;//it is ok, no object is created.

    s = t; // it is ok, s references to t.
  }
  • if two class are under the same package and they are not subclass or superclass each other, they can access protected members each other. If they are under different package and they are not subclass or superclass each other, they can not access protected members of each others.
    package mypackage;

    class A{
      protected int id;

      void increaseAge(B b){
        b.age++;//can access age of B;
      }
    }

    class B{
      protected int age;

      void decreaseId(A a){
        a.id--; // can access id of A
      }
    }
      package package1;
      class A{
        protected int id;

        void increaseAge(B b){
          b.age++; // error, can not access
        }
      }

      package package2;
      class B{
        protected int age;

        void decreaseId(A a){
          a.id--;//error, can not access.
        }
      }

  • With JDK 8 it is possible to add a default implementation to an interface method. (Prior to JDK 8, it is impossible.)

  • JDK 8 also added static interface methods.

  • Beginning with JDK 9, an interface can have private methods.

  • Variables can be declared in interface definitions, they are implicitly static and final.

        interface Reader{
          int id = 1;//it must be initialized, implicitly static and final.

          void printId();
        }

  • All methods and variables in interface are implicitly public.

  • If a class includes an interface but does not fully implemented the methods required by that interface, then class must be declared as an abstract class.

  abstract class Incomplete implements Reader{
    void printHello(){
      System.out.println("Hello");
    }
  }
  • It is possible to use nested interface (member interface) in another class or interface. This kind of nested interface can be declared public, protected or private. Nested interfaces must be accessed by fully qualified name.
  class A{
    public interface NestedInterface{
      boolean isEven(int n);
    }
  }

//B must declared fully qualified name of nested interface (A.NestedInterface)
  class B implements A.NestedInterface{
    @Override
    public boolean isEven(int n){
      return n%2==0;
    }
  }

  class C{
    public static void main(){
      NestedInterface nif = new B();//error, fully qualified name must be used.
      A.NestedInterface nif = new B(); //OK!

      boolean result = b.isEven(22);
    }
  }

  • Primary motivation of default interface methods to provide that interface can be expanded without breaking existing code. Prior to JDK 9, if a new method added to an existing interface, then all the classes would break because no implementation would be found for that new method. So you have to update all the classes.

  • If you want to add default method to an interface, you must use default keyword.

    public interface Reader{
      int getId(); // standard interface methods

      //default method implementation
      default String getFileName(){
        return "File name";
      }
    }

  • If there exist two or more interfaces declared same default method and there is a class which implements those interfaces and does not override the default method, then there would be a compile time error.

  • If an interface I extends interface B and both of them have the same default method. Then a class implements both of them does not need to override the default method.

    interface A{
      default String getHello(){
        return "Hello";
      }
    }

    interface B{
      default String getHello(){
        return "Hello";
      }
    }

    interface I extends B{
      @override
      default String getHello{
        return "Hello from I";
      }
    }

    class C implements A,C{
      //compile time error, no override getHello();
    }

    class D implements A,C{
      @override
      public String getHello(){
        return "D"; // OK. this method takes priority over other default methods.
      }
    }

    class E implements B,I{
      // it is OK. Does not need to implement getHello() method.
    }

    class F{
      public static void main(){
        E e=new E();
        String str = e.getHello(); //str = "Hello from I"
      }
    }
  • With JDK 8, it is possible to add one or more static methods to an interface and it does not need any implementation of the interface. Static interface methods are not inherited by either an implementing class or a sub interface.
  interface A{
    int getId();

    default String getHello(){
      return "Hello";
    }

    // must be defined with method body
    static int getDefaultSalary(){
      return 1000;
    }
  }

  class B implements A{
    public int getId(){
      return -1;
    }
  }

  class D{
    public static void main(){
      int salary = D.getDefaultSalary(); //OK, can be called with interface name directly.
    }
  }
  • Private interface methods can be added into an interface with JDK 9. A private interface method can be called only by a default method or another private method defined by the same interface. A private interface method can not be inherited by a sub interface. The key benefit of the using private methods in an interface is avoiding the code duplications.
  interface A{
    default void diffSquare(int n, int a){
      System.out.println(square(n)-square(a));

    }

    default void printSquare(int n){
        System.out.println(square(n));
    }

    private int square(int n){
        return n*n;
    }
  }
  • All exception types are subclasses of the built-in class Throwable. There are two distinct path immediately under the Throwable class: Exception and Error. Exception class is used for exceptional conditionals that user programs should catch. Error class is used for exceptions that are not expected to be caught by user code. Error type exceptions are used by the Java run time system to indicate errors having to do with the run time environment. Stack overflow is an Error type exception.
